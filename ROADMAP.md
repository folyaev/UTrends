# Roadmap / TODO

Этот файл фиксирует дальнейшие улучшения UTrends. Текущая версия уже пригодна как личный Telegram-бот, но ниже список того, что сделает его удобнее как инструмент для новостного YouTube-канала.

## YouTube-блогеры и новостные каналы

- Добавить fallback за полным описанием YouTube-видео, если в RSS summary нет таймкодов.
- Закешировать описания видео, чтобы не запрашивать YouTube повторно при каждом дайджесте.
- Опционально поддержать YouTube Data API как более стабильный источник `snippet.description`.
- Улучшить извлечение глав: фильтровать рекламу, ссылки, донаты, соцсети и служебные блоки.
- Добавить настройку, какие источники использовать для тем: только title, title+chapters, title+chapters+description.
- Сделать отдельный отчёт по каналам: кто выпустил больше всего роликов, какие темы повторяются у разных авторов, какие темы уникальны.
- Добавить ручной импорт новых YouTube-каналов по handle вида `youtube.com/@name`.
- Показывать предупреждение или отдельную метку для каналов с `banned_ru: true`, если это понадобится в интерфейсе.

## Качество тем и кластеризации

- Улучшить matching русских текстов: морфология, нормальные стоп-слова, более аккуратная обработка имён и организаций.
- Добавить semantic clustering для похожих тем, которые написаны разными словами.
- Снизить ложные совпадения по общим словам вроде `app store`, `путин заявил`, `новости дня`.
- Добавить объяснение кластера: какие слова/каналы/ролики стали причиной объединения.
- Хранить историю качества: какие темы пользователь скрывал, открывал, отслеживал или игнорировал.

## Дайджесты

- Дать пользователю настройки лимитов: сколько повторяющихся тем, одиночных тем и примеров показывать.
- Добавить режим “коротко”: только топ повторяющихся тем без ссылок на все материалы.
- Добавить режим “редакторский”: меньше источников, больше смысла, без шумных одиночных совпадений.
- Сделать отдельную команду для новостных YouTube-каналов, если общий digest станет слишком плотным.
- Добавить персональное расписание дайджеста для разных пользователей.

## RSS и источники

- Хранить историю состояния RSS-источников и автоматически приглушать те, которые регулярно падают.
- Добавить метрики по источникам: сколько новостей пришло, сколько попало в дайджест, сколько было дублей.
- Сделать удобный формат примеров для `feeds.json`, `bloggers.json`, `news_channels.json`, не раскрывая приватные списки.
- Добавить импорт/экспорт источников через админскую команду.

## Watchdog / отслеживаемые темы

- Дать пользователю выбор строгости совпадений для отслеживаемой темы.
- Показывать, почему уведомление сработало: совпавшие слова или похожий заголовок.
- Добавить отдельную команду “только новые совпадения по моим темам”.
- Улучшить хранение длинных тем, чтобы callback-кнопки не обрезали текст.

## Приватность и публикация

- Перед публичным push проверить, что `.env`, `feeds.json`, `bloggers.json`, `news_channels.json`, `trends.db`, backups и личные заметки не попали в Git.
- Перевыпустить Telegram-токен перед публичной публикацией.
- Оставить в репозитории только безопасные example-файлы.
- Добавить короткий чеклист “перед деплоем”.

## Надёжность

- Добавить интеграционные тесты для scheduler-задач, watchdog и Telegram-send rollback.
- Добавить больше тестов на реальные варианты YouTube-таймкодов и описаний.
- Добавить метрики времени сборки дайджеста и количества обработанных источников.
- Параллелить RSS и YouTube-запросы с общим timeout и понятным отчётом по ошибкам.

## Качество кода

- Разнести крупные Telegram handlers из `utrends/bot.py` по feature-модулям.
- Добавить typed DTO для RSS items, YouTube videos и search results.
- Добавить linting/formatting, когда структура кода стабилизируется.
- Сделать отдельный модуль для форматирования Telegram-сообщений дайджеста.
